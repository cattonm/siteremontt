# Патч 5: режим прогулянки в RoomPreview3D (замкнена кімната зі стелею) **Референс:** `room-walk-prototype.html` (корінь пакета). Це еволюція «першої особи» з патча 03: замість look-around на місці — повноцінна хода по кімнаті, зі стелею і всіма 4 стінами. ## Вимоги (з прототипу) 1. **Замикання кімнати.** У режимі прогулянки додаються: передня стіна, права стіна, стеля, плінтуси на нових стінах, тіньовий шов по периметру стелі (якщо обрана «Натяжна» + тіньовий шов). Все — в одній групі `shell`, `shell.visible = mode === 'fp'`. В орбітальному режимі група прихована — макет виглядає як розріз, як зараз. 2. **Камера.** `camera.rotation.order = 'YXZ'`; висота очей EYE = 1.62 м; fov в кімнаті 62 (проти 36 в орбіті). Перехід — lerp позиції+fov за ~850 мс easeInOutQuad (є в патчі 03 як lerpCam). 3. **Огляд.** Драг по канвасу (pointer events) крутить yaw/pitch: `yaw -= dx*0.0042; pitch -= dy*0.0038`, pitch обмежити [-1.25, 1.35] — можна дивитись і на стелю, і на підлогу. OrbitControls у цьому режимі `enabled = false`. 4. **Хода.** WASD/стрілки + віртуальний джойстик (телефон): напівпрозоре коло зліва внизу, pointer capture, вектор нормалізується. Швидкість 1.7 м/с, рух у площині XZ повернутий на yaw. Ледь помітний "bob" кроку: `y = EYE + sin(t*8)*0.008*speed`. 5. **Колізії.** Прості AABB: стіни (margin 0.34 м) + бокси меблів (padding 0.28 м). Push-out по меншій осі проникнення — див. resolveCollisions() у прототипі. Меблі при створенні реєструють свій колайдер. 6. **Світло всередині.** У fp-режимі: hemisphere 0.65→0.95, exposure 1.12→1.3, туман вимкнути. Інакше під стелею темно. 7. **UI.** Дві кнопки-пігулки над канвасом: «Макет» / «Зайти в кімнату». У fp: підказка «W A S D — ходити · тягніть — озиратись» (десктоп), джойстик (тач). Бейдж статусу зліва внизу. ## Інтеграція в React (@react-three/fiber) - `frameloop="demand"` несумісний з ходою — на час fp-режиму перемкнути на `frameloop="always"` (проп на Canvas через стан), назад — 'demand'. - Джойстик і кнопки — звичайний DOM поверх Canvas (absolute), не drei Html. - Хода — useFrame((state, dt) => …) в компоненті ``; keys у ref, joyVec у ref (без setState на кожен кадр!). - Стеля/стіни-shell: рендерити завжди, керувати `visible` — щоб не було стрибка компіляції шейдерів при вході. - Матеріали стелі: «Натяжна матова» roughness 0.95, «сатин» 0.35 + легкий metalness 0.08, «Гіпсокартон» 0.98. Тіньовий шов — чорний бокс 3.5 см по периметру, `visible` залежить від відповіді ceiling_shadow. ## Definition of done - Вхід/вихід плавний, без телепорту; в кімнаті видно стелю зі світильниками (точкові/люстра/LED з відповідей анкети). - На телефоні можна ходити джойстиком і озиратись другим пальцем одночасно (різні pointerId — див. прототип). - Крізь ліжко/шафу пройти не можна; у стіну впертись можна, застрягти — ні.
