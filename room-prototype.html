# Патч 6: редизайн анкети (телефон + планшет) + живий кошторис **Референси в проєкті дизайну:** «Анкета матеріалів.dc.html» (варіанти 2a телефон / 2b планшет — затверджений напрямок), «Прототип анкети.dc.html» (робоча логіка). Токени палітри — з патча 01 (він має бути виконаний першим). ## 6.1 Шапка (App.jsx) Замінити `#top-bar` + `#progress-container`: - **Телефон (<768px):** зліва ← (goBack), по центру назва поточної зони + «крок N з M», справа ⋯ (меню). Під шапкою — сегментований прогрес-бар (по зонах, як зараз), під ним — глобальний TierSwitch (сегмент-контрол Стандарт/Комфорт/Преміум на всю ширину). - **Планшет (≥768px):** одна шапка: назва «Кошторис ремонту» + адреса/площа клієнта зліва, TierSwitch справа. Під нею — зони як хлібні крихти-пігулки (замість бургер-меню): пройдена зона — біла пігулка «✓ Назва» зеленим (#1F7A4D), активна — червона (--accent), майбутня — сіра. Клік = jumpToMenuStep. Бургер-меню на планшеті прибрати. ## 6.2 Липка панель кошторису (App.jsx) Замінити плаваючий `#live-cart` (чорна пігулка) на **закріплену панель на всю ширину** внизу, над nav-bar (або об'єднати з nav-bar в один блок): - Рядок сум: `Роботи {work} ₴ · Матеріали від {mat_min} ₴` + якщо поточний крок — кімната у візуалізаторі, третя сума по цій кімнаті (є в liveBreakdown.rooms). - Справа лінк «Деталі ▸» — відкриває існуючий bottom-sheet (isCartOpen). - Кнопки ← і «Далі» — в цій самій панелі (див. 2a/2b: ← квадратна 52px, «Далі» червона на решту ширини). - background: var(--card); border-top: 1px solid var(--line); box-shadow: 0 -8px 24px rgba(20,18,14,0.06). Без donut на панелі — donut лишається в bottom-sheet. ## 6.3 Ціни на картках НЕ показувати Рішення власника: жодних цін на картках вибору. Ціни — лише в липкій панелі та в «Деталі ▸» / Summary. ## 6.4 Survey.jsx — стилі под нову палітру Класи лишаються, оновлюється CSS (index.css): - `.card`: border-radius 16px, border 1.5px var(--line); selected → border var(--accent), фон rgba(194,37,29,0.06/0.08), у selected картки з фото — бейдж-галочка 20px у правому верхньому куті фото. - `.pill`: border-radius 100px, min-height 44px (тач-таргет!). - `.tier-btn`: active → фон var(--accent), білий текст (зараз, ймовірно, синій). - `.card-grid`: `grid-template-columns: repeat(auto-fill, minmax(160px, 1fr))` — на телефоні 2 колонки, на планшеті 4 (замість фіксованих 2). - Бейджі статусу групи (візуалізатор, compact-режим): відповіли → зелене коло ✓; обов'язкова без відповіді → пігулка «Необхідно обрати» (accent на rgba-фоні). Логіка вже є (isQuestionAnswered/REQUIRED_GROUPS) — це лише стиль. ## 6.5 Фікс семантики rough_plaster (⚠ баг) `questions.js`: текст питання — «Чи потрібна машинна штукатурка стін (чорнова)?» (Так/Ні), але бекенд (remontnikuav/calculator.py) додає штукатурку при відповіді **«Ні»** — стара семантика «чи вже зроблена». Зараз клієнт, що відповідає «Так, потрібна», НЕ отримує штукатурку в кошторисі, і навпаки. Виправлення на боці фронтенда (бекенд не чіпаємо): повернути питанню стару семантику тексту: ```js { id: "rough_plaster_done", zone: "ЧОРНОВІ / ЗАГАЛЬНЕ", text: "Чи оштукатурені стіни (чорнова штукатурка вже є)?", type: "cards", options: [{label:"Так, вже є",val:"Так"},{label:"Ні, потрібна штукатурка",val:"Ні"}] }, ``` Альтернатива (краща, але потребує деплою бекенда): лишити текст «Чи потрібна…» і інвертувати умову в calculator.py. Обрати ОДИН варіант, не обидва. ## 6.6 Планшетна сітка контенту На ≥768px обмежити контент `max-width: 1000px; margin: 0 auto`. `.animated-step h3` → 24px. Питання типу multiselect_complex на планшеті — в 2 колонки (grid). ## Definition of done - Телефон: шапка зі степером, глобальний тумблер, картки 2 колонки, панель сум закріплена знизу, «Далі»/← у ній. - Планшет: крихти зон замість бургера, картки 4 колонки, та сама панель на всю ширину. - Перемикання глобального рівня миттєво перераховує суми (існуючий live_calc — без змін). - Питання про штукатурку більше не працює навпаки.
